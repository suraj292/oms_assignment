# Git Commit Examples

These are realistic commit messages you might use for this Phase 0 setup.

## Initial Setup Commits

### Backend Setup
```bash
git add backend/
git commit -m "chore: initialize Laravel 11 API with Sanctum auth

- Set up Laravel 11 as REST API
- Configure Sanctum for token authentication  
- Add BaseApiController with response helpers
- Create folder structure for controllers, services, and requests
- Configure MySQL database connection
- Set up public storage link"
```

### Frontend Setup
```bash
git add frontend/
git commit -m "chore: bootstrap Vue 3 SPA with router and pinia

- Initialize Vue 3 + TypeScript with Vite
- Install and configure Vue Router 4
- Install and configure Pinia for state management
- Set up Axios client with auth interceptor
- Create auth store with token persistence
- Configure path aliases (@/ for src/)
- Add basic folder structure"
```

### Documentation
```bash
git add README.md FOLDER_STRUCTURE.md SETUP.md .gitcommit-examples.md
git commit -m "docs: add comprehensive setup and structure documentation

- Add main README with setup instructions
- Document folder structure patterns and scaling recommendations
- Include quick start guide and verification steps
- Add example commit messages for reference"
```

## Alternative Granular Commits

If you prefer smaller, more focused commits:

### Backend - Step by Step

```bash
# 1. Initial Laravel install
git add backend/
git commit -m "chore: initialize Laravel 11 project"

# 2. Sanctum setup
git add backend/config/sanctum.php backend/database/migrations/*sanctum*
git commit -m "feat: install and configure Laravel Sanctum

- Add Sanctum package
- Publish Sanctum config and migrations
- Configure stateful domains for local development"

# 3. API configuration
git add backend/bootstrap/app.php backend/routes/api.php
git commit -m "feat: configure Laravel as API-only application

- Enable API routes in bootstrap/app.php
- Add Sanctum middleware to API routes
- Create basic health check endpoint"

# 4. Base controller
git add backend/app/Http/Controllers/Api/
git commit -m "feat: add BaseApiController with response helpers

- Create standardized JSON response methods
- Add successResponse and errorResponse helpers
- Set up API controller namespace"

# 5. Environment config
git add backend/.env.example
git commit -m "chore: configure environment for MySQL and Sanctum

- Update .env.example with MySQL defaults
- Add Sanctum stateful domains configuration
- Document required environment variables"
```

### Frontend - Step by Step

```bash
# 1. Initial Vite + Vue install
git add frontend/
git commit -m "chore: initialize Vue 3 project with Vite and TypeScript"

# 2. Core dependencies
git add frontend/package.json frontend/package-lock.json
git commit -m "chore: install vue-router, pinia, and axios

- Add Vue Router 4 for routing
- Add Pinia for state management
- Add Axios for HTTP requests"

# 3. Router setup
git add frontend/src/router/ frontend/src/views/
git commit -m "feat: configure Vue Router with lazy loading

- Set up router with history mode
- Create HomeView component
- Enable lazy loading for routes"

# 4. Pinia store
git add frontend/src/stores/
git commit -m "feat: create auth store with token persistence

- Set up Pinia auth store
- Add token management with localStorage
- Implement logout functionality"

# 5. API client
git add frontend/src/api/
git commit -m "feat: configure Axios client with interceptors

- Create Axios instance with base URL
- Add request interceptor for auth tokens
- Add response interceptor for 401 handling"

# 6. Path aliases
git add frontend/vite.config.ts frontend/tsconfig.app.json
git commit -m "chore: configure path aliases for cleaner imports

- Add @ alias pointing to src/ directory
- Update Vite and TypeScript configs
- Enable absolute imports from src/"

# 7. Main app integration
git add frontend/src/main.ts frontend/src/App.vue
git commit -m "feat: integrate router and pinia into main app

- Register Pinia and Router plugins
- Update App.vue to use RouterView
- Clean up default Vite template"

# 8. Environment config
git add frontend/.env.example frontend/.gitignore
git commit -m "chore: configure environment variables

- Add .env.example with API URL
- Update .gitignore for environment files
- Document required environment variables"
```

## Commit Message Format

Following conventional commits format:

### Types
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation only
- `chore:` - Maintenance (dependencies, config)
- `refactor:` - Code restructuring
- `test:` - Adding tests
- `style:` - Code style/formatting

### Structure
```
<type>: <short summary>

<optional body with details>
<bullet points for multiple changes>

<optional footer>
```

### Examples

**Good commits:**
```bash
feat: add user authentication endpoints

- Create login and register controllers
- Add request validation for auth
- Implement JWT token generation

Closes #123
```

```bash
fix: resolve CORS issue with frontend requests

Updated sanctum stateful domains to include
the frontend dev server URL.
```

```bash
chore: update dependencies to latest versions

- Laravel 11.47.0
- Vue 3.5.13
- Pinia 2.3.0
```

**Avoid:**
```bash
# Too vague
git commit -m "updates"
git commit -m "fix stuff"

# Too long in summary
git commit -m "add user authentication with login and register and password reset and email verification"

# No context
git commit -m "changes"
```

## Workflow Tips

### Before Committing
```bash
# Check what's changed
git status

# Review changes
git diff

# Stage specific files
git add path/to/file

# Or stage all changes
git add .

# Commit with message
git commit -m "type: summary"
```

### After Setup
```bash
# View commit history
git log --oneline

# See what's in the last commit
git show

# Amend last commit (if needed)
git commit --amend
```

## Branching Strategy (For Future)

When you start feature work:

```bash
# Create feature branch
git checkout -b feature/user-authentication

# Work on feature...
git add .
git commit -m "feat: implement user registration"

# More commits...
git commit -m "feat: add login endpoint"
git commit -m "test: add auth integration tests"

# Merge back to main
git checkout main
git merge feature/user-authentication
```

## Summary

- Use clear, descriptive commit messages
- Follow conventional commits format
- Keep commits focused and atomic
- Write messages that explain "why" not just "what"
- Use present tense ("add" not "added")
- Reference issues/tickets when applicable
